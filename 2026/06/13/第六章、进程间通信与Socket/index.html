第六章、进程间通信与Socket

发表于 2026-06-13 分类于 Linux系统编程

本章总结的重点是：同一主机关注 IPC 机制，不同主机关注 socket、字节序、UDP/TCP 通信流程。

一、进程为什么需要通信

进程拥有独立的地址空间，这是稳定性和安全性的基础。但现实程序经常需要协作：

  • 一个进程采集数据，另一个进程处理数据。
  • 父进程创建子进程执行任务，并接收结果。
  • 多个服务进程共享状态。
  • 不同主机上的程序通过网络交换消息。

这就需要 IPC，也就是 Inter-Process Communication，进程间通信。

通信场景分成两类：

  • 同一主机：管道、FIFO、System V IPC、共享内存等。
  • 不同主机：socket、UDP、TCP。

二、同一主机：管道

管道必须凑齐读写双方。管道由内核提供，是单工通信机制，并且具有自同步特性：快的一方会被慢的一方限制。

三、System V IPC

XSI / System V IPC 的关键概念：

  • key：通信双方通过同一个 key 找到同一个 IPC 对象。
  • ftok()：根据路径和项目 ID 生成 key。
  • xxxget：创建或获取对象。
  • xxxop：操作对象。
  • xxxctl：控制对象，例如删除。

三类典型对象：

  • Message Queues：消息队列。
  • Semaphore Arrays：信号量数组。
  • Shared Memory：共享内存。

四、主动端与被动端

  • 主动端：先发包的一方。
  • 被动端：先收包的一方，通常先运行并等待数据。

在消息队列中，如果主动方先运行并发送数据，消息可能会被内核缓存起来，缓存能力受系统限制影响。网络编程中也有类似思想：服务器一般先启动监听，客户端再连接或发送。

五、项目练习：俄罗斯方块

  • 图形问题：VT 图形码以光标为单位绘图，framebuffer 以像素为单位着色。
  • 输入设备设置：tcgetattr()、tcsetattr()。
  • 并发：下落逻辑和键盘输入响应。

这个练习非常适合串联系统编程知识：

  • 终端控制用于绘制界面。
  • 非阻塞输入或多线程用于同时处理下落和键盘。
  • 定时器或线程 sleep 用于控制节奏。
  • 结构体用于表示方块、地图和状态。

如果扩展成多进程版本，还可以用管道或共享内存把输入、渲染和游戏逻辑拆开。

八、UDP 通信流程

报式套接字流程：

被动端：

  1. 获取 socket。
  2. 绑定 socket 地址。
  3. 收发消息。
  4. 关闭 socket。

主动端：

  1. 获取 socket。
  2. 可以绑定地址，也可以让系统自动分配。
  3. 收发消息。
  4. 关闭 socket。

九、TCP 通信流程

流式套接字流程：

主动端：

  1. 获取 socket。
  2. 给 socket 取得地址。
  3. 发起连接。
  4. 收发消息。
  5. 关闭连接。

被动端：

  1. 获取 socket。
  2. 绑定地址。
  3. 设置为监听模式。
  4. 接受连接。
  5. 收发消息。
  6. 关闭连接。

总结

IPC 的选择取决于通信双方在哪里、数据量多大、是否需要同步、是否跨主机。管道简单，消息队列有边界，共享内存快但需要同步，socket 则把通信扩展到网络。掌握这些机制后，就能从单进程程序走向多进程服务和网络服务。
